异步数据交互基本上网站开发都会用到，只要使用jquery的$.ajax，即使是一个完全不懂XMLHttpRequest的前端也能立刻写出一个异步的数据交互。那么现在问题是，ajax究竟需要考虑哪些情况？

如果是页面跳转，那么所有的数据都是一次性得出的，不存在局部刷新的可能。所以就不需要前端开发人员去考虑异常处理，用户自己就会F5全页面刷新。

以实际情况为例，用户点击一个链接之后，异步获取内容刷新显示区域。那么至少需要这样写：

$.ajax({url:'xxx.php'})
.success(function(){ /* TODO */ });

如果要求对失败的情况也做一些事情，那么可以增加一个fail方法：

$.ajax({url:'xxx.php'})
.success(function(){ /* TODO */ })
.fail(function(){ /* TODO*/ });

正如开始所说的，网络是一个不稳定的东西，所以会因为各种原因超时。所以还应该把超时的情况写进去：

$.ajax({ url:'xxx.php', timeout:5000 })
.success(function(){ /* TODO */ })
.fail(function(){ /* TODO*/ });

到这里常规的ajax请求就算完成了，所以总结一个完整的交互至少应该考虑三种情况：

  1. 成功 success
  2. 失败 fail
  3. 超时 timeout
