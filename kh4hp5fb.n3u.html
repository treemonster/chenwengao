我们在给树莓派3B型装完系统之后，会遇到可能64G的卡只用了3G多一点，这是因为树莓派默认没有把整个存储空间拓展到整张TF卡中。这时候我们可以通过人为的把存储空间拓展到整张TF卡上。

我的环境

raspbian系统，装在树莓派3 B型 的64G TF卡。

解决步骤 查资料发现所有人都有共同问题，是由于默认没有扩展存储空间到整张卡，必须通过命令行重建分区（命令行下的 fdisk 工具）。

首先打开命令行：Ctrl + Alt + T

输入命令检查当前存储空间大小：

df -h 查看第二分区起始地址，要记下来数字：

cat /sys/block/mmcblk0/mmcblk0p2/start 接着使用 fdisk 操作磁盘：

sudo fdisk /dev/mmcblk0 (下面进入了fdisk工具，根据引导选择操作，并且按下回车)

依次输入下面的命令，空行代表按回车 (注意：是应答分别输入命令，不是一次性输入一句命令) d 2 n p 2

(输入刚刚记下的第二分区起始地址数字) (最后一个选择直接回车，默认即可)

w

最后的 w 代表写入，这个操作我试过无数次，每次都有红色的错误提示，大意是说硬盘被占用。很多人卡在这一步，其实根本不用理会，直管继续！

然后要重启一下计算机：

sudo reboot 重启回来以后，执行：

sudo resize2fs /dev/mmcblk0p2 接着再查看存储空间，看看变大没有：

df -h 到这里就结束了~
