git clone时，出现SSL certificate problem: self signed certificate in certificate chain SSL证书的问题，可以忽略证书继续重新执行 git config --global http.sslVerify false

但是，这种情况很大可能是目标服务器受到了中间人攻击，所以最好先浏览器打开看看网站证书是不是有问题。2020年3月27号早上就是这个情况，国内的dns服务器被攻击了，因此正确操作不是忽略证书校验，而是挂国外代理流量访问。

我错误的使用了上面提到的方法，因此我在挂流量之后首选操作修改github密码，希望不会因此丢失账户。以后切记
